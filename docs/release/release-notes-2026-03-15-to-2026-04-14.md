## Executive Summary

This 30-day period represents the most transformative batch of changes to the easy-claude framework to date. Four major architectural shifts define this release: (1) the introduction of a **code graph system** — a zero-config, multi-language structural intelligence layer that traces function calls, service boundaries, and dependency flows; (2) a **shared-protocol elimination** where 15+ standalone protocol files were deleted and their content inlined directly into each skill via SYNC tags, eliminating indirection and prompt-skip failures; (3) **adversarial mindset upgrades** across 16 review skills via steel-man, inversion, and pre-mortem protocols; and (4) a **direct doc injection** mechanism where hooks read and embed reference-doc content into AI context rather than issuing "MUST READ" instructions that the AI would skip. These changes collectively make Claude Code more structurally aware, harder to fool, and more consistent in following project patterns.

## 1. Code Graph System (New Major Feature)

### What It Is

A new Python-based structural intelligence engine (`code_graph/`) that parses multi-language codebases into a SQLite graph database and provides Claude Code with deep structural awareness: call chains, class hierarchies, event flows, API boundaries, and CQRS dispatch patterns.

**Commits:** `dd3a4e3`, `69f56b6`, `8887b38`, `1fb26d6`, `c5d1289`

### Engine Components Added

| Module                             | Purpose                                                                                |
| ---------------------------------- | -------------------------------------------------------------------------------------- |
| `code_graph/parser.py`             | Multi-language AST parser (C#, TypeScript, JS, Python, Go, Java, Ruby) via tree-sitter |
| `code_graph/graph.py`              | SQLite-backed graph builder with node/edge schema                                      |
| `code_graph/cli.py`                | CLI entry point — `trace`, `query`, `connections`, `search`, `batch-query` commands    |
| `code_graph/api_connector.py`      | Zero-config REST API edge detection from URL patterns                                  |
| `code_graph/api_patterns.py`       | Per-framework API pattern registry                                                     |
| `code_graph/implicit_connector.py` | Bare-call resolution — resolves unqualified calls via import disambiguation            |
| `code_graph/incremental.py`        | Git-aware incremental rebuild (only changed files)                                     |
| `code_graph/mermaid_exporter.py`   | Mermaid diagram export from graph subsets                                              |
| `code_graph/tools.py`              | Claude Code tool wrappers                                                              |
| `code_graph/models.py`             | Node/Edge dataclasses, edge kind registry                                              |
| `code_graph/descriptions.py`       | Human-readable descriptions of edge kinds                                              |

### Edge Kinds Supported

`CALLS`, `IMPLEMENTS`, `INHERITS`, `DEFINES`, `IMPORTS`, `PRODUCES_EVENT`, `CONSUMES_EVENT`, `MESSAGE_BUS`, `TRIGGERS_COMMAND_EVENT`, `CQRS_DISPATCH` (new in this period), `API_ENDPOINT`, and auto-discovered kinds from DB.

### Post-Build Enhancements (Auto-Run)

After parsing, three enhancement passes run automatically:

1. **Call Noise Filtering** — Per-language built-in suppression (e.g., `map`, `push`, `forEach` in JS; `print`, `len` in Python). Configurable per-project via `project-config.json → graphSettings.callNoiseFilter`.
2. **Bare Call Resolution** — Resolves unqualified calls by scanning import declarations; eliminates ambiguous edges.
3. **Auto-discover Edge Kinds** — Reads edge kind set from DB instead of a hardcoded list, making the system self-extending.

The `.claude/skills/shared/` directory (15+ protocol files) was completely deleted:

```

shared/estimation-framework.md                   ← Deleted

shared/module-detection-keywords.md              ← Deleted

shared/red-flag-stop-conditions-protocol.md      ← Deleted
shared/refinement-dor-checklist-protocol.md      ← Deleted
shared/scaffold-production-readiness-protocol.md ← Deleted
shared/two-stage-task-review-protocol.md         ← Deleted
shared/ui-wireframe-protocol.md                  ← Deleted
shared/understand-code-first-protocol.md         ← Deleted
shared/web-research-protocol.md                  ← Deleted
```

### Why

Shared protocol files created a **reference indirection problem**: skills pointed to protocol docs, but AI would skip the read, resulting in unenforced protocols. The indirection also made skills opaque — you couldn't read a skill and know its full behavior.

### How They Were Replaced

Content was inlined directly into each skill's `SKILL.md` via **SYNC tags**:

```markdown
<!-- SYNC:evidence-based-reasoning-protocol START -->

...protocol content...

<!-- SYNC:evidence-based-reasoning-protocol END -->
```

A new `sync-protocols` skill (`/sync-protocols`) and `shared/sync-inline-versions.md` tracker manage version consistency across the inlined copies.

### Impact on 130+ Skills

Every skill that referenced shared protocols was updated to carry the full protocol inline. Bulk update covered 130+ skill files in two commits.

## Addendum — 2026-04-14

### Subagent Context Injection: Full 13-Hook Paging Refactor

**Problem:** Claude Code enforces a per-hook `additionalContext` size limit (~9,000 chars). The previous `subagent-init.cjs` monolith injected all context in a single hook, silently truncating large documents:

- CLAUDE.md (~13KB) — lost Graph Intelligence, Skill Activation, Doc Lookup sections
- development-rules.md (~18KB) — lost architecture and naming convention sections
- coding patterns (~36KB) — severely truncated, missing most patterns

No error was raised — the hook appeared to succeed while silently dropping content.

**Solution:** Replace the single `subagent-init.cjs` with 13 sequential SubagentStart hooks, each guaranteed ≤9KB output. A new shared library `subagent-context-builders.cjs` centralizes all context builder functions and the `splitContentIntoPart()` paginator.

### 13-Hook Sequence

| #   | Hook                                 | Content                                              | Output (verified) |
| --- | ------------------------------------ | ---------------------------------------------------- | ----------------- |
| 1   | `subagent-init-identity.cjs`         | Agent identity, plan context, language, rules        | 1,240 chars       |
| 2–6 | `subagent-init-patterns-p[1-5].cjs`  | Coding patterns (5-part split, ~36KB total)          | 3,104–8,557 chars |
| 7–9 | `subagent-init-dev-rules-p[1-3].cjs` | development-rules.md (3-part split, ~18KB total)     | 1,356–8,575 chars |
| 10  | `subagent-init-lessons.cjs`          | Shared lessons from prior sessions                   | 1,548 chars       |
| 11  | `subagent-init-ai-mistakes.cjs`      | AI mistake prevention bullets (fabrication, halluc.) | 8,194 chars       |
| 12  | `subagent-init-context-guard.cjs`    | Context-window-overflow guard (fires 12th of 13)     | 743 chars         |
| 13  | `subagent-init-todos.cjs`            | Active parent todo list (silent when no todos)       | 0–varies          |

### Paging Implementation

**`splitContentIntoPart(content, partNumber, totalParts)`** — O(n) line-boundary paginator in `subagent-context-builders.cjs`:

- Splits content into `totalParts` equal slices at line boundaries (never mid-line)
- Each part includes a `Part X/Y: lines A–B of C (N%)` header for observability
- Hooks exit silently (no output) when their slice is empty — enables graceful shrinkage if source files decrease in size

**`emitSubagentContext(lines)`** — shared output wrapper:

- Wraps output in `{ hookSpecificOutput: { additionalContext: [...] } }` format
- Silent exit (no stdout) when `lines` array is empty — fail-open, no noise

**Overflow guard (patterns-p5):** If any document exceeds its allocated parts, the final part appends: `[OVERFLOW] N lines (~M chars) not injected. Read {file} from line X.`

### PreCompact State-Capture + Post-Compact Re-Verify Warning

**Problem:** When Claude Code compacts the session context, all phase-state knowledge is lost. The AI would resume trusting the summary's claims about "completed" phases without re-verifying. This caused silent continuation errors (e.g., applying a phase-2 fix when phase-1 was never persisted).

**Solution:** Two-part hook enhancement:

**Part 1 — `write-compact-marker.cjs` (PreCompact):** Now captures `git status --short` at compact time and stores it in the marker file as `compactState: { gitStatus, warningShown: false }`. On a clean working tree (`gitStatus = ""`), the truthy guard omits `compactState` (no spurious objects written). Fails silently in non-git directories.

**Part 2 — `prompt-context-assembler.cjs` (UserPromptSubmit):** On the first prompt after a compact, `buildPostCompactWarning()` reads the marker file, injects a one-time re-verify warning block showing the git status snapshot, then sets `warningShown: true` to prevent repeat injection on subsequent prompts (dedup via disk-persisted flag across process boundaries).

**New shared constant:** `SESSION_ID_DEFAULT = 'default'` added to `lib/ck-paths.cjs` and imported by both hooks — eliminates the silent key-mismatch bug where the writer used `|| 'default'` and the reader passed `|| null`, preventing the warning from ever firing when `session_id` was absent from the hook payload.

**Warning block format (injected once per compact):**

```
## ⚠️ CONTEXT COMPACTED — Re-verify before continuing
Context was compacted since your last prompt. The summary may not reflect what actually persisted in the environment.
**Git status at compact time:** [snapshot]
**MANDATORY:** Before resuming any in-progress task: run git status, re-read files, treat "completed" claims as UNVERIFIED HYPOTHESES.
```
